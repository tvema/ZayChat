# Закрытый Мессенджер (Документация проекта)

## Требования к проекту
### 1. Система инвайтов (приглашений)
* Регистрация возможна только по уникальной пригласительной ссылке от уже существующего пользователя.
* Ссылка является одноразовой, содержит уникальный код и ID пользователя-отправителя.
* В базе данных жестко фиксируется реферальная связь — кто именно пригласил нового пользователя.

### 2. Регистрация и Профиль
* Обязательные поля: Аватар, Никнейм, Имя, Фамилия, E-mail, Мобильный телефон, Пароль.
* Аватар: При загрузке открывается редактор с возможностью масштабирования и кадрирования (выбора области). Итоговое изображение сохраняется строго в размере 256x256 пикселей.
* Предусмотрена возможность редактирования профиля (включая смену аватара) в будущем.

### 3. Поиск и Контакты (Окружение)
* Глобальный поиск: возможность искать других уже зарегистрированных пользователей системы (например, по нику, имени или телефону).
* Добавление в контакты: найденных пользователей можно добавлять в свой личный список контактов ("своё окружение"), чтобы начать с ними переписку.

### 4. Интерфейс приложения (Главный экран)
* Левая колонка (Сайдбар):
  * Верхняя часть: Карточка профиля текущего пользователя (аватар, имя, доступ к настройкам и генерации инвайтов).
  * Нижняя часть: Список добавленных контактов ("окружение").
* Правая колонка (Основная):
  * Окно чата с выбранным собеседником из списка контактов (история сообщений, поле ввода, кнопка отправки).
  * **Смайлы и реакции:** Возможность добавлять смайлы (реакции) к конкретным сообщениям (постам).
  * **Крупные смайлы:** В теле самого чата смайлы отображаются крупнее обычного текста.
  * **Кастомизация:** Цвета "пузырей" сообщений (для текущего пользователя и для оппонента) вынесены в отдельный конфигурационный файл для удобной настройки.

## Архитектура и Техническая реализация
- **Frontend**: Next.js App Router, Tailwind CSS, Lucide React, Framer Motion
- **Backend**: Express.js (кастомный сервер), Socket.IO, SQLite (better-sqlite3)
- **База данных**: SQLite (`chat.db`) для хранения пользователей, инвайтов, контактов, сообщений и реакций.
- **Хранение файлов**: Локальная файловая система (`public/uploads`) для аватаров, WebRTC + IndexedDB для файлов в чате.
- **Клиентская библиотека**: `react-easy-crop` для обрезки фото.

## Процесс передачи файлов P2P (WebRTC)
1. Отправитель выбирает файл. Он сохраняется в его локальной `IndexedDB`.
2. Отправитель посылает JSON-сообщение с метаданными через WebSocket.
3. Получатель принимает сообщение. Если файла нет в его `IndexedDB`, он отправляет событие `webrtc:request_file` отправителю.
4. Отправитель получает запрос, создает `RTCPeerConnection` и открывает `RTCDataChannel`.
5. Сигналинг WebRTC (Offer/Answer/ICE) происходит через WebSocket.
6. Как только DataChannel открыт, отправитель разбивает файл на части (по 16 КБ) и отправляет их, учитывая `bufferedAmount` для предотвращения переполнения буфера.
7. Получатель собирает части файла, сохраняет в `IndexedDB` и отображает файл.

## План действий
### Этап 1: Подготовка инфраструктуры и Базы Данных
1. Установка необходимых зависимостей: `express`, `socket.io`, `better-sqlite3`, `bcryptjs`, `jsonwebtoken`, `multer`, `react-easy-crop`, `emoji-picker-react`.
2. Настройка кастомного сервера Node.js (`server.ts`) для работы Next.js вместе с WebSockets.
3. Проектирование и инициализация базы данных SQLite (таблицы: `users`, `invites`, `contacts`, `messages`, `reactions`).

### Этап 2: API и Авторизация (Backend)
1. Реализация генерации и проверки инвайт-кодов.
2. Создание эндпоинтов для регистрации (с загрузкой и сохранением аватара) и логина.
3. Настройка JWT-авторизации для защиты маршрутов и сокет-соединений.
4. Эндпоинты для поиска пользователей и добавления их в контакты.

### Этап 3: Frontend - Регистрация и Профиль
1. Создание страницы входа (Login).
2. Создание страницы регистрации по инвайт-коду.
3. Интеграция `react-easy-crop` для загрузки, масштабирования и обрезки аватара до 256x256 пикселей.

### Этап 4: Frontend - Главный экран и Чат
1. Создание конфигурационного файла `chat-theme.config.ts` для настройки цветов пузырей сообщений.
2. Верстка левой панели (Сайдбар): профиль пользователя, кнопка генерации инвайта, список контактов.
3. Верстка правой панели (Окно чата): история переписки, поле ввода.
4. Интеграция WebSockets на клиенте для отправки и получения сообщений в реальном времени.

### Этап 5: Фишки чата (Смайлы и Реакции)
1. Настройка рендеринга сообщений: увеличение размера одиночных смайлов в тексте.
2. Добавление UI для выбора и прикрепления реакций (смайлов) к конкретным сообщениям.
3. Синхронизация реакций через WebSockets.

### Версия 1.2.2 - Исправления видеозвонков WebRTC для мобильных устройств
* **Совместимость с мобильными браузерами:** Исправлена критическая проблема, из-за которой видеозвонки приводили к черному экрану в мобильных браузерах (особенно в Safari на iOS).
* **Нативная обработка MediaStream:** Выполнен переход от ручного добавления дорожек к использованию нативного объекта браузера `event.streams[0]`, что значительно повышает надежность рендеринга в браузерах на базе WebKit.

* **Ручной перезапуск видео:** Добавлена функция ручного переопределения — нажатие (тап) в любом месте экрана во время активного звонка принудительно перепривязывает видеопоток (`srcObject = null`, затем повторное назначение), что гарантирует перерисовку кадра, если браузер завис.

### Версия 1.2.1
* **Единая активная сессия:** Внедрено строгое правило "одно устройство на аккаунт". Вход в приложение с нового устройства или вкладки браузера теперь автоматически завершает сеанс на всех ранее активных устройствах. Это предотвращает конфликты WebSocket-соединений и гарантирует доставку сообщений на правильное активное устройство.

* **Миниатюры прикрепленных файлов:** Улучшен интерфейс ответа на сообщения с файлами. Теперь вместо скучной надписи "File attachment" в блоке предпросмотра ответа и в самом процитированном сообщении отображается компактная миниатюра картинки (или иконка для других типов файлов).

### Версия 1.1.3 - Асинхронный баг WebRTC DataChannel и исправление CSS
- **Исправление ошибки**: Исправлен критический баг, появившийся в v1.1.2, из-за которого передача файла зависала на "Waiting for peer...". Ошибка была вызвана состоянием гонки, когда `currentFileId` сбрасывался в `null` сообщением `end` до того, как промис `saveFile` разрешался, что приводило к тихому сбою при отправке события `file-downloaded`.
- **Исправление ошибки**: Изменен CSS превью изображений с `object-cover` на `object-contain`. Изначальная жалоба пользователя на "обрезанные изображения" на самом деле была связана с тем, что CSS визуально обрезал изображение под контейнер, а не с потерей данных WebRTC.

### Версия 1.1.2 - Исправление порядка сообщений WebRTC DataChannel
- **Исправление ошибки**: Исправлен баг, из-за которого изображения/файлы иногда обрезались (например, нижняя часть изображения отсутствовала или была серой). Это было вызвано известной проблемой WebRTC, когда строковые сообщения (например, сигнал `end`) могут обгонять бинарные чанки в DataChannel. Теперь получатель проверяет `receivedSize >= totalSize`, чтобы гарантировать получение всех байтов перед сохранением файла в IndexedDB, вместо того чтобы полагаться на сообщение `end`.
